Http VS Https

✅ What is HTTP?

  • HyperText Transfer Protocol (HTTP) is the foundation of data communication on the web.
  • It allows the transfer of text, images, videos, and other web content between a client (browser) and a server.
  • Stateless protocol → Each request is independent, meaning no session data is stored between connections.
  • Default Port: 80

 

✅ What is HTTPS?

  • HyperText Transfer Protocol Secure (HTTPS) is the secured version of HTTP.
  • Uses SSL/TLS encryption to protect data transmitted between the client and server.
  • Ensures confidentiality, integrity, and authentication of the data being exchanged.
  • Default Port: 443

✅ Why Use HTTPS?

  • Protects user data (passwords, credit card details, personal info).
  • Prevents MITM (Man-in-the-Middle) attacks where hackers intercept data.
  • Builds trust and credibility with users.
  • Required for modern web standards (e.g., browser security warnings on non-HTTPS sites).
